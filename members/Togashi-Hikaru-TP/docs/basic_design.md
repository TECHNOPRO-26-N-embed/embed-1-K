```
実装時の状態 enum 案（Arduino/C++）

enum SystemState {
STATE_BOOT = 0, // 初期化中
STATE_IDLE, // 待機中
STATE_INPUT, // 入力中
STATE_CALCULATING, // 計算中
STATE_SHOW_RESULT, // 結果表示中
STATE_ERROR, // エラー中
STATE_RESETTING // リセット中
};
```

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| （例）待機中 | センサー値を100msごとに監視する | 緑LED：500msで点滅 |
| 待機中 | 数値入力を待機している | ブザー：オフ |
| 入力中 | アナログスティックで数値を入力中 | ブザー：入力確定時に短く鳴動 |
| 計算中 | 入力された数値で計算を実行中 | ブザー：計算完了時に鳴動 |
| 結果表示中 | 計算結果を表示中 | ブザー：結果表示時に鳴動 |
| リセット中 | システムを初期状態に戻す | ブザー：リセット時に鳴動 |
|  |  |  |

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| （例）LED点滅 | 500ms | **millis** | ボタン監視と並行するため |
| （例）センサー読み取り | 100ms | **millis** | 常時監視が必要なため |
| アナログスティック入力 | 100ms | **millis** | 入力の反応速度を確保するため |
| タクトスイッチ入力 | 100ms | **millis** | 入力の反応速度を確保するため |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

## 4. 異常系・エラー処理の方針

| 異常ケース | 発生条件 | 検知方法 | 対応方針 |
|:--|:--|:--|:--|
| チャタリング | ボタンを押したとき | 前回入力から50ms未満の入力を検出 | デバウンス処理でスキップ |
| センサー異常値 | センサーが誤読したとき | 値が仕様範囲外（例: 0 or 400cm超） | その値を無視・前回値を使用 |
| アナログノイズ | アナログ入力を読んだとき | 閾値付近で値が揺れる | 閾値に±10〜20のバッファを設ける |
| 0除算 | 計算時に0で割ろうとしたとき | 演算子が除算で、被除数が0 | エラーメッセージを表示し、計算を中止、リセット |
| スティックの異常値 | スティックが故障しているとき | 値が常に0や最大値のまま変わらない | エラーメッセージを表示し、入力を無効化 |
| 電源障害 | 電源が不安定なとき | 電圧センサーで電圧を監視 | 電圧が規定値を下回ったらエラーメッセージを表示し、動作を停止 |
|  |  |  |  |
