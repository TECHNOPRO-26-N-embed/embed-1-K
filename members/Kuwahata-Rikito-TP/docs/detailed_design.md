### `setup()` — 初期化処理

```
【処理の流れ】
1. ピンモードを設定する
  - センサー系ピン → INPUT/OUTPUT
  - 出力系ピン（ブザー・表示）→ OUTPUT

2. ライブラリの初期化（使うものだけ）
  - 例: TM1637初期化

3. Serial.begin(9600)（デバッグ用）

4. 起動確認（任意）: ブザーを短く鳴らして停止
```

**↓ 自分の setup() を設計してください**
```
【処理の流れ】
1. ピンモードを設定する
  - PIN_TRIG: OUTPUT, PIN_ECHO: INPUT
  - PIN_BUZZER: OUTPUT
  - PIN_LED_OPT: OUTPUT（ライトの機能を使う場合）
2. 4桁7セグ（TM1637）を初期化し、初期表示（0000）を出す
3. Serial.begin(9600)を実行する（任意のデバッグ用）
4. 起動確認としてブザーを100msだけ鳴らして停止する
5. 状態変数を初期化する
  currentState=0,（1: 距離判定中、2: 通知中、3: 異常監視） 
  distanceZone=2,（0: 近距離、1: 中距離、2: 遠距離）
  distanceMm=0,（現在計測された距離（mm単位）
  prevValidDistanceMm=0,（直前の有効な距離値（mm単位）を保持する変数）
```

### `loop()` — メインループ

> ※ loop() は「状態ごとに何をするか」だけ書く。細かい処理は各関数に任せる。

```
【処理の流れ】

＜毎ループ実行すること＞
  - 入力を読む（readDistanceMm() などを呼ぶ）
  - 現在時刻を取得: now = millis()

＜currentState が 0（待機中）のとき＞
  - センサー値を監視する
  - 検知条件を満たしたら → currentState = 1

＜currentState が 1（動作中）のとき＞
  - メイン処理を行う
  - 終了条件を満たしたら → currentState = 2

＜currentState が 2（完了）のとき＞
  - 完了表示をする
  - 再開条件を満たしたら → currentState = 0

＜currentState が 3（エラー）のとき＞
  - エラー表示をする / リセットを待つ
```

**↓ 自分の loop() を設計してください**
```
【処理の流れ】

＜毎ループ実行すること＞
  - now = millis() を取得する
    （現在時刻が更新され、周期判定に使えるようになる）
  - applyThresholdOptional() を呼び、しきい値設定を反映する
    （不整合なしならしきい値をそのまま使用。不整合があれば安全値へ補正される）
    （しきい値を固定値で運用する場合は呼び出し不要。可変にする設計時のみ必要）

＜currentState が 0（待機中）のとき＞
  - now - lastMeasureMillis >= 100ms のとき readDistanceMm() を呼ぶ
    （距離が更新され、有効値なら distanceMm と prevValidDistanceMm が最新化される）
  - 距離が有効値なら currentState = 1（距離判定中）

＜currentState が 1（距離判定中）のとき＞
  - classifyDistanceZone(distanceMm) を呼んで distanceZone を更新
    （近・中・遠のいずれかに分類され、通知モードが確定する）
  - 異常値なら currentState = 3（異常監視）
  - 正常なら currentState = 2（通知中）

＜currentState が 2（通知中）のとき＞
  - updateBuzzerOutput(distanceZone) を呼ぶ
    （距離区分に応じた音パターンへ切り替わる）
  - updateDisplayOutput(distanceMm) を呼ぶ
    （表示器が最新距離に更新される）
  - doLedOptional(distanceZone) を呼ぶ
    （LEDを使う構成なら、距離区分に応じた点灯・点滅に更新される）
    （LED部品を搭載しない構成では不要。光による追加通知を行う構成で必要）
  - doSerialReportOptional() を呼ぶ
    （シリアルモニタに距離と区分が出力され、動作確認しやすくなる）
    （本番運用でシリアル出力が不要な場合は省略可能。デバッグ時は有効）
  - 次周期の再判定のため currentState = 1

＜currentState が 3（異常監視）のとき＞
  - ブザー停止、表示は prevValidDistanceMm を維持
    （誤通知を防ぎつつ、最後に正常だった値を表示して挙動を安定化する）
  - 次周期で再計測し、有効値に復帰したら currentState = 1
```

### `readDistanceMm()` — HC-SR04から距離を取得して保存する

**basic_design.md 2-2 との対応：** F01 距離を周期的に測定

**引数：** なし

**戻り値：** `int`（距離mm）

```
【処理の流れ】
1. Trigに10usパルスを出力し、Echoのパルス幅を取得する
2. パルス幅から距離(mm)へ変換する
3. 値が有効範囲（1〜4000mm）なら distanceMm と prevValidDistanceMm を更新して返す
4. 値が範囲外なら prevValidDistanceMm を返す

【エラー・異常ケース】
- Echoタイムアウト時は異常値として扱い、currentState=3へ遷移させる
```

---

### `classifyDistanceZone(int distanceMm)` — 距離を近/中/遠へ分類する

**basic_design.md 2-2 との対応：** F02 3段階に分類

**引数：** `distanceMm`（`int`）: 判定対象の距離

**戻り値：** `uint8_t`（0:近 1:中 2:遠）

```
【処理の流れ】
1. ヒステリシスを考慮して境界判定値を決める（±20mm）
2. distanceMm <= nearThresholdMm なら 0（近）
3. nearThresholdMm < distanceMm <= midThresholdMm なら 1（中）
4. midThresholdMm < distanceMm なら 2（遠）

【エラー・異常ケース】
- 異常値入力時は前回の distanceZone を維持する
```

---

### `updateBuzzerOutput(uint8_t zone)` — 距離区分に応じた音を出す

**basic_design.md 2-2 との対応：** F03 区分別にブザー出力

**引数：** `zone`（`uint8_t`）: 0近/1中/2遠

**戻り値：** `void`

```
【処理の流れ】
1. zone=0（近）のとき: 高音を連続出力する
2. zone=1（中）のとき: 100ms ON / 100ms OFF の断続音を出す
3. zone=2（遠）のとき: 低音の短音を周期的に出す

【エラー・異常ケース】
- 不正zoneの場合は noTone() を実行して停止する
```

---

### `updateDisplayOutput(int distanceMm)` — 距離を4桁7セグへ表示する

**basic_design.md 2-2 との対応：** F04 距離を数値表示

**引数：** `distanceMm`（`int`）: 表示する距離

**戻り値：** `void`

```
【処理の流れ】
1. 表示値を0〜9999の範囲に丸める
2. TM1637表示形式に変換する
3. 4桁7セグへ表示を更新する

【エラー・異常ケース】
- 表示更新失敗時はシリアルへエラー出力し、前回表示を維持する
```

---

### `doSerialReportOptional()` — 距離と音モードをシリアル出力する

**basic_design.md 2-2 との対応：** A01 距離と音モードをシリアル表示（任意）

**引数：** なし

**戻り値：** `void`

```
【処理の流れ】
1. distanceMm と distanceZone を文字列化する
2. "dist=xxxxmm zone=x" の形式で出力する
3. 出力周期を必要に応じて間引く

【エラー・異常ケース】
- Serial未接続時は出力処理をスキップする
```

---

### `applyThresholdOptional()` — しきい値定数を反映する

**basic_design.md 2-2 との対応：** A02 しきい値を定数で調整（任意）

**引数：** なし

**戻り値：** `void`

```
【処理の流れ】
1. nearThresholdMm と midThresholdMm の大小関係を確認する
2. 不整合があれば安全値（100, 300）へ補正する
3. 判定関数で参照する定数を更新する

【エラー・異常ケース】
- 不正設定時は補正してシリアルへ警告を出す
```

---

### `doLedOptional(uint8_t zone)` — 距離区分に応じたLED表示を行う

**basic_design.md 2-2 との対応：** A03 光の表現追加（任意）

**引数：** `zone`（`uint8_t`）: 0近/1中/2遠

**戻り値：** `void`

```
【処理の流れ】
1. zone=0ならLEDを高速点滅
2. zone=1ならLEDを中速点滅
3. zone=2ならLEDを低速点滅または消灯

【エラー・異常ケース】
- 不正zoneではLED消灯を維持する
```

### 3-2. millis() を使ったタイマー管理

```
【考え方】
  「前回実行した時刻」を記録しておき、「今の時刻 − 前回時刻 ≥ 周期」なら実行する。

【処理の流れ（例: LED点滅）】
  1. now = millis()
  2. now - lastMillis_LED >= LED_INTERVAL かどうか確認
  3. 条件を満たした場合: LEDのON/OFFを切り替え、lastMillis_LED = now
  4. 条件を満たさない場合: 何もしない（次のループで再チェック）

【自分のシステムで millis() を使う処理】
  - 距離読み取り: 100ms周期（lastMeasureMillis）
  - 中距離断続音: 100ms ON/OFF（lastBeepToggleMillis）
  - 表示更新: 100ms周期（計測と同周期で更新）
```

### Q1: 実装上の問題確認

> 「この詳細設計書に書いた関数と処理フローをもとに Arduino でコードを書きます。バグになりやすい箇所・処理の抜け・型の問題はありますか？」

**AIの回答（要約）：**
- 関数分割は妥当で、計測・分類・出力が独立している。
- 注意点はHC-SR04の異常値処理と、TM1637更新とtone()の周期干渉。
- loop()でdelayを使わない方針は適切。

**対応した内容：**
- 異常値時に prevValidDistanceMm を使うフォールバックを明記。
- 中距離断続音を millis() で制御する設計に固定。
