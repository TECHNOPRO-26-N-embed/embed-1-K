# 配線ガイド（Unit1: 距離で音が変わるお知らせブザー）

このファイルは、段階デモを実施するための配線手順をまとめたものです。

- 段階1: ブザーのみ
- 段階2: ブザー + 超音波センサー
- 段階3: ブザー + 超音波センサー + 4桁7セグ(TM1637)
- 段階4: ブザー + 超音波センサー + 4桁7セグ(74HC595)

対象コード:
- `src/test_01_buzzer_only.ino`
- `src/test_02_sensor_buzzer.ino`
- `src/test_03_sensor_buzzer_display.ino`
- `src/test_04_sensor_buzzer_74hc595_display.ino`

---

## 0. 事前準備

### 使用部品
- Arduino本体（UNO想定）
- パッシブブザー 1個
- HC-SR04 超音波センサー 1個
- TM1637 4桁7セグ表示器 1個
- ブレッドボード、ジャンパ線

### 共通注意
- すべての部品で GND を共通化してください。
- 配線変更前は、Arduino の USB を抜いてから作業してください。
- ピン番号はコードと完全一致させてください。

### UNO上で使う場所（先に全体像）
- DIGITALヘッダ（D0〜D13）
  - D4: TM1637 CLK
  - D5: TM1637 DIO
  - D6: ブザー信号
  - D9: HC-SR04 Trig
  - D10: HC-SR04 Echo
- POWERヘッダ
  - 5V: HC-SR04 VCC と TM1637 VCC へ給電
  - GND: ブザー -, HC-SR04 GND, TM1637 GND へ接続
- ANALOGヘッダ（A0〜A5）
  - 今回は未使用（接続不要）

### 配線のやり方（ブレッドボード使用時）
- UNOの `5V` をブレッドボードの `+` ラインへ1本つなぐ。
- UNOの `GND` をブレッドボードの `-` ラインへ1本つなぐ。
- 各部品の `VCC` は `+` ラインへ、各部品の `GND` は `-` ラインへつなぐ。
- 各信号線（D4/D5/D6/D9/D10）は、UNOのDIGITALヘッダから部品の信号端子へ直接つなぐ。

## 1. 段階1（ブザーのみ）

使用コード: `src/test_01_buzzer_only.ino`

### 配線
- ブザー `+`（または信号端子） -> UNOの `DIGITAL D6`
- ブザー `-` -> UNOの `POWER GND`

### どこに挿すか（UNO基準）
- `D6` は UNOの上側にある `DIGITAL` 列の `~6` ピン。
- `GND` は UNOの `POWER` 列にある `GND` ピン。
- 2端子ブザーの場合は `+` と `-` を逆にすると鳴らないことがあるので極性を確認する。

### 動作確認
- 約0.8秒ごとに「高音 -> 中音 -> 低音」が順番に切り替わる。
- 音が鳴らない場合は次を確認する。
  - ブザーの極性（+/-）
  - `D6` の刺し間違い
  - GND未接続

## 2. 段階2（ブザー + HC-SR04）

使用コード: `src/test_02_sensor_buzzer.ino`

### 追加配線（段階1に加える）
- HC-SR04 `VCC` -> UNOの `POWER 5V`
- HC-SR04 `GND` -> UNOの `POWER GND`
- HC-SR04 `Trig` -> UNOの `DIGITAL D9`
- HC-SR04 `Echo` -> UNOの `DIGITAL D10`

### どこに挿すか（UNO基準）
- `D9` と `D10` は UNO上側の `DIGITAL` 列。
- `5V` と `GND` は UNO左下付近の `POWER` 列。
- HC-SR04の端子並びは一般的に `VCC / Trig / Echo / GND`（基板印字を必ず確認）。
- `Trig` と `Echo` を逆にすると計測できない。

### 動作確認
- 近距離（約10cm以内）: 高音が連続で鳴る。
- 中距離（約10cm超〜30cm以下）: 断続音になる。
- 遠距離（約30cm超）: 短音 + 長め無音になる。

### トラブル時の確認
- 反応しない場合:
  - Trig/Echo の逆接続
  - VCC/GND の逆接続
  - 対象物が小さすぎる/斜めで反射しにくい
- 常に同じ音の場合:
  - センサー前に手を置いて距離を大きく変えて再確認
  - 電源不足（USBハブ経由の場合は直挿しで試験）

## 3. 段階3（ブザー + HC-SR04 + TM1637）

使用コード: `src/test_03_sensor_buzzer_display.ino`

### 追加配線（段階2に加える）
- TM1637 `VCC` -> UNOの `POWER 5V`
- TM1637 `GND` -> UNOの `POWER GND`
- TM1637 `CLK` -> UNOの `DIGITAL D4`
- TM1637 `DIO` -> UNOの `DIGITAL D5`

### どこに挿すか（UNO基準）
- `D4` と `D5` は UNO上側の `DIGITAL` 列。
- `CLK` と `DIO` は名前が似ているため、コードと同じ対応で接続する。
  - `CLK` -> `D4`
  - `DIO` -> `D5`
- TM1637モジュールの端子は一般に `CLK / DIO / VCC / GND` だが、製品ごとに並びが異なる場合があるので印字を確認する。

### 動作確認
- ブザー挙動は段階2と同じ。
- 4桁7セグに距離(mm)が表示される。
- 例: 200mm のとき `0200` 表示。

### トラブル時の確認
- 表示が出ない場合:
  - `CLK` と `DIO` の差し間違い
  - TM1637 ライブラリ未導入
- 表示が不安定な場合:
  - GND共通不足
  - ジャンパ線接触不良

## 4B. 74HC595版（表示テスト済み状態から追加配線）

使用コード: `src/test_04_sensor_buzzer_74hc595_display.ino`

この章は、以下の状態から始めることを前提にしています。

- すでに 74HC595 + 4桁7セグ表示のテストで点灯確認済み
- 74HC595は Elegoo のサンプル通り
  - `DS -> D8`
  - `STCP(LATCH) -> D9`
  - `SHCP(CLOCK) -> D10`
- 4桁選択は `D2 / D3 / D4 / D5`

### 4B-1. 追加で使うピン（競合なし）

- ブザー信号: `D6`
- HC-SR04 Trig: `D11`
- HC-SR04 Echo: `D12`
- 電源:
  - HC-SR04 `VCC -> 5V`
  - HC-SR04 `GND -> GND`
  - ブザー `- -> GND`

### 4B-2. 作業前チェック

- ArduinoのUSBを抜く。
- すでに動いている表示系配線（`D2,D3,D4,D5,D8,D9,D10`）は触らない。
- ブレッドボードの `+` / `-` 電源レールが途中で分断されるタイプか目視確認する。
  - 分断される場合は、ジャンパ線で分断部を橋渡しする。

### 4B-3. 追加配線手順（1本ずつ）

1. UNO `GND` からブレッドボード `-` レールへ1本つなぐ（未接続なら必須）。
2. UNO `5V` からブレッドボード `+` レールへ1本つなぐ（未接続なら必須）。
3. ブザーの `+` を UNO `D6` へつなぐ。
4. ブザーの `-` を `-` レールへつなぐ。
5. HC-SR04 `VCC` を `+` レールへつなぐ。
6. HC-SR04 `GND` を `-` レールへつなぐ。
7. HC-SR04 `Trig` を UNO `D11` へつなぐ。
8. HC-SR04 `Echo` を UNO `D12` へつなぐ。

### 4B-4. HC-SR04端子の注意

- 端子順はモジュール印字を最優先で確認する。
- 一般的には `VCC / Trig / Echo / GND` だが、並びが異なる製品もある。
- `Trig` と `Echo` を逆接続すると距離が読めない。

### 4B-5. ブザーの注意

- 2端子ブザーは極性を確認する（`+` と `-`）。
- 3端子モジュール型の場合は `S(信号)` を `D6` に接続し、`VCC/GND` を各電源へ接続する。

### 4B-6. 追加後の動作確認

1. `src/test_04_sensor_buzzer_74hc595_display.ino` を書き込む。
2. 表示が4桁で更新されることを確認する。
3. 手をセンサー前で動かし、表示値(mm)が変わることを確認する。
4. 距離に応じてブザー音パターンが変わることを確認する。

### 4B-7. トラブル時の確認

- ブザーが鳴らない:
  - `D6` の刺し間違い
  - ブザー極性逆
  - GND未共通
- センサーが反応しない:
  - `D11(Trig)` / `D12(Echo)` の逆接続
  - `VCC/GND` の誤配線
- 表示が急に不安定になった:
  - 追加作業時に `D8/D9/D10` か `D2/D3/D4/D5` が抜けていないか再確認
  - 電源レール分断でHC-SR04やブザーに給電できているか確認

## 6. デモ実施の順番（推奨）

1. `test_01_buzzer_only.ino` 書き込み -> ブザー単体確認
2. `test_02_sensor_buzzer.ino` 書き込み -> 距離連動音確認
3. `test_03_sensor_buzzer_display.ino` 書き込み -> 距離表示を含めた統合確認
4. `test_04_sensor_buzzer_74hc595_display.ino` 書き込み -> 74HC595版の統合確認
